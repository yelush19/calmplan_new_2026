# 📊 הנחיות להקמת לוחות Monday.com עבור CalmPlan

## 🎯 סקירה כללית
המערכת דורשת מספר לוחות ב-Monday.com לניהול מלא של המשימות והתהליכים. מסמך זה מרכז את כל ההנחיות להקמת הלוחות הנדרשים.

---

## 📋 לוח 1: התאמות חשבונות

### פרטי הלוח
- **שם הלוח:** `התאמות חשבונות`
- **סוג הלוח:** `Main Board` (לוח ראשי)
- **תיאור:** `מעקב וניהול התאמות בנקים, אשראי וחשבונות פנימיים ללקוחות`

### קבוצות (Groups)
1. `התאמות לחודש הנוכחי`
2. `התאמות לחודש הבא`
3. `הושלמו` (או `ארכיון`)

### עמודות (Columns) - חובה להקפיד על השמות והסוגים!

| שם העמודה | סוג העמודה | הערות/אפשרויות |
|------------|------------|----------------|
| `שם לקוח` | Text | עמודת ה-Item Name (ברירת מחדל) |
| `חשבון` | Text | שם החשבון המותאם (לדוגמה: "בנק לאומי - עו"ש") |
| `סוג התאמה` | Dropdown | אפשרויות: `בנק`, `אשראי`, `הנהלת חשבונות`, `סליקה` |
| `תקופה` | Text | פורמט: `YYYY-MM` (לדוגמה: `2024-06`) |
| `תאריך יעד` | Date | תאריך יעד לביצוע ההתאמה |
| `סטטוס` | Status | תוויות: `ממתין לחומרים`, `בביצוע`, `בעיה`, `הושלם` |
| `כרטיס הנה"ח` | Text | מספר כרטיס הנהלת חשבונות |
| `הערות` | Long Text | הערות רלוונטיות |
| `קישור ללקוח` | Connect Boards | קישור ללוח `Clients` |
| `ID_משימה_CalmPlan` | Text | **אל תשני ידנית!** |
| `ID_חשבון_CalmPlan` | Text | **אל תשני ידנית!** |

## 🔧 הגדרות נוספות לכל הלוחות

### 1. הרשאות
- **בעלים:** את
- **צוות:** בני המשפחה (לפי הצורך)
- **צפייה בלבד:** אורחים (אם רלוונטי)

### 2. אוטומציות מומלצות ב-Monday
- **התראה על דדליינים:** 3 ימים לפני תאריך יעד
- **העברה אוטומטית:** משימות שהושלמו לקבוצת "ארכיון"
- **צביעת שורות:** לפי דחיפות או סטטוס

### 3. תצוגות (Views) מומלצות
- **תצוגת לוח (Board):** לניהול יומיומי
- **תצוגת גאנט:** לתכנון לטווח ארוך
- **תצוגת לוח שנה:** למשימות עם תאריכים
- **תצוגת טבלה:** לניתוח נתונים
