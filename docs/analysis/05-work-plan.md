# תוכנית עבודה מדורגת — CalmPlan

**מטרה:** להפוך את CalmPlan ממערכת עם הרבה חלקים מנותקים → למערכת עובדת מוכנה למסירה למשרד הנהלת חשבונות.

**עיקרון מנחה:** לא לשבור מה שעובד. כל שלב נבנה, נבדק, ורק אז עוברים הלאה.

---

## שלב א' — דף הבית עובד באמת (1-2 שבועות)

**למה קודם:** דף הבית הוא מה שבעלת המשרד רואה כל בוקר. אם הוא עובד — היא סומכת על המערכת.

| # | משימה | פירוט | מורכבות |
|---|--------|--------|---------|
| 1.1 | קיפול סקשנים ב-Home | סקשנים מתקפלים כברירת מחדל. "היום" פתוח. "באיחור" מציג Badge עם מספר | קלה |
| 1.2 | מצב שקט — 5 משימות | ברירת מחדל: 5 משימות מובילות. כפתור "הצג הכל" פותח רשימה בדיאלוג | קלה |
| 1.3 | חיבור Task Insights | להציג 3-4 תובנות מ-Cascade Engine: "3 לקוחות מחכים למע"מ", "שכר בהמתנה" | קלה |
| 1.4 | חיבור SmartNudge | דחיפה חכמה אחת בראש העמוד — "הגיע הזמן לטפל ב..." לפי עדיפות | קלה |
| 1.5 | כפתורי אנרגיה | 3 כפתורים: מלא/בינוני/נמוך — מסננים את המשימות המוצגות | קלה |
| 1.6 | חיבור MoodTracker | צ'ק-אין בוקרי: "איך את מרגישה?" → משפיע על סוג המשימות | קלה |

**תוצאה:** דף בית שמציג רק מה שרלוונטי, לא מציף, ומותאם למצב הרוח והאנרגיה.

## כללי ברזל

1. **לא לשבור מה שעובד** — כל שינוי נבדק לפני שעוברים הלאה
2. **עיקרון הצגה מותאמת** — דף הבית ו"מה לעשות עכשיו" מציגים 5 פריטים מובילים כברירת מחדל. דשבורדים מקצועיים (שכר, הנה"ח, מיסים, לקוחות) מציגים את הרשימה המלאה — כי שם צריך לראות תמונה מלאה
3. **אפס אדום** — פלטת צבעים ללא אדום/ורוד (Zero Panic)
4. **ADHD-first** — כל החלטה עיצובית נבחנת: "האם זה מפחית עומס או מוסיף?"
5. **הלקוחה קודם** — בעלת המשרד משתמשת במערכת. לא מוסיפים פיצ'רים על חשבון יציבות
