## רקע והבנת הבעיה

### מה לא עובד עכשיו
1. המערכת דוחפת הכל לדדליין — "כדור שלג" של עומס ביום האחרון
2. אין הבדל בין משימה שאפשר להתחיל עכשיו למשימה שתלויה בלקוח
3. מורכבות מוגדרת ברמת לקוח — אבל לקוח יכול להיות ננו בשכר וכבד בהנהלת חשבונות
4. הדשבורד מציג רשימה שטוחה — אין היררכיה, אין תעדוף, אין כיוון
5. ימי שישי/שבת/חגים לא נלקחים בחשבון בתכנון

### הפתרון — 3 שכבות

```
שכבה 1: הגדרות (פעם אחת)
  → עומס לכל תהליך + לקוח
  → מקור נתונים לכל תהליך
  → ימים חסומים

שכבה 2: תכנון גמיש (פעם בחודש בערך)
  → פריסת עבודה חכמה אחורה מהדדליין
  → התחשבות בעומס + ימים חסומים

שכבה 3: דשבורד בוקר (כל יום)
  → מה עושים היום — ברור ומדויק
  → אפשרות שינוי מהיר
```

## חלק ב׳ — שדות חדשים בטבלת תהליכים/תחומים

### שדה: מקור נתונים
כל תהליך מקבל הגדרה של מאיפה מגיעים הנתונים.

```sql
-- בסופהבייס: ALTER TABLE task_types (או טבלת התהליכים הקיימת)
data_source  TEXT  -- 'self' | 'client' | 'mixed'
```

| ערך | משמעות | השפעה על תכנון |
|-----|---------|---------------|
| `self` | יש לי גישה עצמאית לנתונים | נכנס לתכנון מיד, אפשר לפרוס |
| `client` | חייב לחכות שהלקוח ישלח | נשאר "ממתין" עד קבלה, לא נכנס לפריסה |
| `mixed` | חלק ממני, חלק מהלקוח | נכנס לתכנון חלקי + אינדיקטור "ממתין להשלמה" |

### דוגמאות לערכים
```
שכר                → client  (תלוי בנתוני שעות/שינויים מהלקוח)
מע"מ               → self    (יש גישה למערכת החשבונות)
הנהלת חשבונות     → mixed   (בנק = self, חשבוניות = client)
ביטוח לאומי        → self    (גישה למערכת)
דוח שנתי           → mixed
פנסיות             → client
```

### ימים חסומים (ברירת מחדל)
```typescript
// src/constants/blockedDays.ts

export const DEFAULT_BLOCKED_DAYS = {
  weekly: [5, 6],  // שישי (5) ושבת (6) — index 0 = ראשון
  
  // חגים קבועים — יש לעדכן בתחילת כל שנה
  holidays: [
    // 2026
    '2026-04-13', // ערב פסח
    '2026-04-14', // פסח
    '2026-04-20', // שביעי של פסח
    '2026-05-02', // ערב יום העצמאות
    '2026-05-03', // יום העצמאות
    '2026-06-01', // ערב שבועות
    '2026-06-02', // שבועות
    '2026-09-20', // ערב ראש השנה
    '2026-09-21', '2026-09-22', // ראש השנה
    '2026-09-29', // ערב יום כיפור
    '2026-09-30', // יום כיפור
    '2026-10-04', // ערב סוכות
    '2026-10-05', // סוכות
    '2026-10-11', // הושענא רבה
    '2026-10-12', // שמיני עצרת
  ]
};

// פונקציה: האם יום מסוים זמין לעבודה
export function isWorkDay(date: Date, override = false): boolean {
  if (override) return true;  // "אין ברירה" — מאפשר עבודה בחג
  
  const dayOfWeek = date.getDay();
  const dateStr = date.toISOString().split('T')[0];
  
  if (DEFAULT_BLOCKED_DAYS.weekly.includes(dayOfWeek)) return false;
  if (DEFAULT_BLOCKED_DAYS.holidays.includes(dateStr)) return false;
  
  return true;
}

// פונקציה: ספור ימי עבודה זמינים בין שני תאריכים
export function countWorkDays(from: Date, to: Date): number {
  let count = 0;
  const current = new Date(from);
  while (current <= to) {
    if (isWorkDay(current)) count++;
    current.setDate(current.getDate() + 1);
  }
  return count;
}
```

### אלגוריתם הפריסה
```typescript
// src/utils/workloadSpread.ts

interface SpreadInput {
  deadline: Date;
  totalWorkload: number;      // יחידות עומס (מחושב לפי לקוח + תחום)
  dataSource: 'self' | 'client' | 'mixed';
  clientMaterialsExpected?: Date; // מתי הלקוח אמור לשלוח
}

interface SpreadResult {
  canStart: Date;              // מתי אפשר להתחיל
  recommendedStartDate: Date;  // מתי כדאי להתחיל
  dailyLoad: number;           // עומס מומלץ ליום
  warningDaysBuffer: number;   // כמה ימי חיץ נשארו
  status: 'ok' | 'tight' | 'overloaded';
}

export function calculateSpread(input: SpreadInput): SpreadResult {
  const today = new Date();
  const workDaysAvailable = countWorkDays(today, input.deadline);
  
  // אם תלוי בלקוח — מחשב מתאריך קבלת חומרים
  const effectiveStart = input.dataSource === 'client' && input.clientMaterialsExpected
    ? input.clientMaterialsExpected
    : today;

  const effectiveWorkDays = countWorkDays(effectiveStart, input.deadline);
  const dailyLoad = input.totalWorkload / effectiveWorkDays;
  
  // חיץ מומלץ: 20% מזמן העבודה
  const bufferDays = Math.floor(effectiveWorkDays * 0.2);
  
  return {
    canStart: effectiveStart,
    recommendedStartDate: subtractWorkDays(input.deadline, 
      Math.ceil(input.totalWorkload / DAILY_CAPACITY) + bufferDays),
    dailyLoad,
    warningDaysBuffer: bufferDays,
    status: dailyLoad > DAILY_CAPACITY * 1.5 ? 'overloaded' 
          : dailyLoad > DAILY_CAPACITY ? 'tight' 
          : 'ok'
  };
}

// קיבולת יומית ממוצעת — ניתן לכוונון
const DAILY_CAPACITY = 8; // יחידות עומס
```

### לוגיקת מיון ל"עכשיו" / "אחר כך"

```typescript
function classifyTask(task: Task, today: Date): 'now' | 'later' | 'waiting' | 'done' {
  if (task.status === 'done') return 'done';
  
  // ממתין — תלוי בלקוח ועדיין לא הגיעו חומרים
  if (task.dataSource === 'client' && !task.materialsReceived) return 'waiting';
  
  const daysUntilDeadline = differenceInDays(task.deadline, today);
  const spread = calculateSpread(task);
  
  // עכשיו — דדליין תוך 48 שעות, או שהמערכת תכננה להתחיל היום
  if (daysUntilDeadline <= 2) return 'now';
  if (isSameDay(spread.recommendedStartDate, today)) return 'now';
  
  // אחר כך — הכל השאר
  return 'later';
}
```

## סיכום — מה המערכת יודעת לעשות אחרי היישום

```
✅ מחשבת עומס לכל לקוח לפי תחום (לא עומס גורף)
✅ יודעת אילו תהליכים מחכים ללקוח ואילו לא
✅ לא מתכננת עבודה בשישי/שבת/חגים
✅ מזהירה מוקדם כשיש עומס חריג
✅ דשבורד בוקר אומר "התחילי כאן" — לא "הנה הכל"
✅ גמישות מלאה — כל תכנון ניתן לשינוי יומי
✅ ממתין ללקוח = לא מופיע בתכנון עד שמגיע
```
